~25% of exam
💻
Binary & Number Systems
Binary (Base-2):
Each position: 2⁷  2⁶  2⁵  2⁴  2³  2²  2¹  2⁰
Value:          128  64   32   16   8    4    2    1

Example: 10110101 = 128+32+16+4+1 = 181
Hexadecimal (Base-16):
0-9, A=10, B=11, C=12, D=13, E=14, F=15
Each hex digit = 4 binary digits
FF = 1111 1111 = 255
Bits and Bytes:
1 byte = 8 bits
1 KB = 1,024 bytes ≈ 10³
1 MB = 1,024 KB ≈ 10⁶
1 GB = 1,024 MB ≈ 10⁹
Overflow:
If a value exceeds what n bits can store, overflow error occurs.
With 8 bits: max unsigned = 255. 256 → wraps to 0.
Data Compression
Lossless:
No data lost. Original can be perfectly reconstructed. Used for text, code, ZIP files. (Run-length encoding: AAABBCC → 3A2B2C).
Lossy:
Some data permanently removed. Smaller file, cannot recover original. Used for images (JPEG), audio (MP3), video (MP4). Human perception exploited — removes data we don't notice.
Metadata:
Data about data. A photo's metadata: date, GPS location, camera model, resolution. Metadata can reveal more than the data itself (NSA phone metadata controversy).

1. Convert binary 1101 to decimal.
A) 11
B) 13
C) 14
D) 15
B: 13.
1101: 1×8 + 1×4 + 0×2 + 1×1 = 8+4+0+1 = 13. Always write out the place values: 8,4,2,1 for a 4-bit number.
2. Which is an example of LOSSY compression?
A) ZIP file
B) Text file encoding
C) MP3 audio compression
D) Run-length encoding
C: MP3.
MP3 permanently removes audio data that human hearing is less sensitive to (high frequencies, sounds masked by louder sounds). The original audio cannot be recovered. ZIP (A) is lossless — your files are perfectly recovered.

AP CSP List Trap — 1-Indexed!
AP Pseudocode lists start at index
1
, NOT 0.
myList ← [10, 20, 30] → myList[1] = 10, myList[2] = 20, myList[3] = 30.
This is the #1 pseudocode mistake on the AP exam — especially if you know Python (0-indexed) or Java (0-indexed).
LENGTH(myList) returns the count of elements. Last element = myList[LENGTH(myList)].
💻
Algorithms — Key Concepts
Linear (Sequential) Search
Check each element one by one. Works on ANY list. O(n) worst case. Simple but slow for large lists.
Binary Search
Only works on SORTED lists. Compare to middle, eliminate half. O(log n). Much faster — but requires sorted data.
Undecidable Problems
Some problems cannot be solved by any algorithm — halting problem. No matter how much computing power, there is no algorithm that always correctly answers 'will this program halt?'
Heuristics
Approximate solutions for problems where exact solution is impractical. Not guaranteed optimal, but good enough in reasonable time. Used in: GPS routing, chess AI, spam filters.

1. In AP Pseudocode, myList ← [5, 10, 15, 20]. What is myList[3]?
A) 5
B) 10
C) 15
D) 20
C: 15.
AP Pseudocode is 1-indexed. myList[1]=5, myList[2]=10, myList[3]=15, myList[4]=20. This trips up students who know Python (0-indexed). On the exam, always remember: AP Pseudocode starts at 1.
2. Binary search REQUIRES that the data be:
A) Stored in a 2D array
B) Sorted
C) Numerical
D) Larger than 100 elements
B: Sorted.
Binary search works by comparing the target to the middle element and eliminating half the list. This only works if the data is sorted — otherwise you can't know which half to keep.
Big Idea: Networks
Internet & Protocols
~10–13% of exam

How the Internet Works
Concept
Definition
Key Detail
IP Address
Unique identifier for every device on a network
IPv4: 32-bit (4 octets, e.g. 192.168.1.1). IPv6: 128-bit (more addresses)
DNS
Domain Name System — translates URLs to IP addresses
'Phone book of the internet': google.com → 142.250.80.46
HTTP/HTTPS
Protocol for web communication
HTTPS uses TLS/SSL encryption. HTTP is unencrypted.
TCP/IP
Transmission Control Protocol / Internet Protocol
TCP ensures reliable, ordered delivery. Packets may take different routes.
UDP
User Datagram Protocol
Faster but unreliable (no delivery guarantee). Used for video streaming, gaming.
Packet Switching
Data split into packets, sent independently, reassembled
Each packet may take a different path. More efficient and fault-tolerant than circuit switching.
Redundancy
Multiple paths between nodes in a network
If one path fails, data reroutes. The internet was designed for this resilience.
📝 Unit Quiz
0/0 correct
Reset
1. Why does packet switching make the internet more fault-tolerant than circuit switching?
A) Packets are smaller than circuits
B) Packets can take multiple routes, so if one path fails, data can reroute through alternative paths
C) Packet switching is faster
D) Packets are encrypted by default
B is correct.
In circuit switching (like old telephone calls), one dedicated path is established for the entire communication — if it breaks, the call drops. Packet switching breaks data into independently routed packets, so network failures only affect some packets, which can be rerouted or re-requested.
Create PT
Performance Task — 6-Row Rubric
30% of AP Score
💻
Create PT Requirements
What You Submit
1.
Program code
— your program file(s). Must include at least one procedure you wrote.
2.
Video
— 1 minute max showing your program running and its input/output.
3.
Written responses
— 4 prompts (3A–3D) submitted in the AP Digital Portfolio.
💻
The 6-Row Rubric — Row by Row
Row
Points
What to Include
Row 1: Program Purpose & Function
1
Describe what the program does (purpose), what it accepts as input, and what it produces as output. Reference the VIDEO.
Row 2: Data Abstraction
1
Show a LIST being used to store/access data. Explain how the list stores multiple values and why a list was necessary vs. individual variables.
Row 3: Managing Complexity
1
Explain how the list MANAGES COMPLEXITY — how it would be more complex without the list. Don't just say it stores data — explain what would change in the code without it.
Row 4: Procedural Abstraction
1
Show a PROCEDURE with a parameter that affects functionality. Explain what the procedure does, what the parameter represents, and how it's called.
Row 5: Algorithm Implementation
1
Show an ALGORITHM inside your procedure that includes: SEQUENCING + SELECTION (if/conditional) + ITERATION (loop). Trace through the algorithm with a specific input.
Row 6: Testing
1
Describe TWO DIFFERENT test cases with specific inputs, expected outputs, and actual outputs. Tests should check different conditions (e.g., edge case vs. normal case).
Create PT Mistakes That Lose Points
Row 2:
Showing a list but using it just as a display — the list must actually store values that the program uses for functionality.
Row 3:
'The list stores all the values' — not enough. Explain what CODE would look like without the list (parallel variables) and why that increases complexity.
Row 4:
Calling a built-in function (DISPLAY, APPEND) doesn't count. Must be a STUDENT-CREATED procedure with a PARAMETER.
Row 5:
The algorithm must have all three: sequence, selection, AND iteration. One missing = 0.
Row 6:
Two test cases with the SAME condition = 0. Tests must cover DIFFERENT scenarios.

<h3 class="sub"><span class="sub-icon">💻</span>Binary & Number Systems</h3><div class="code-block"><span class="hl">Binary (Base-2):</span>
Each position: 2⁷  2⁶  2⁵  2⁴  2³  2²  2¹  2⁰
Value:          128  64   32   16   8    4    2    1

Example: 10110101 = 128+32+16+4+1 = 181

<span class="hl">Hexadecimal (Base-16):</span> 0-9, A=10, B=11, C=12, D=13, E=14, F=15
Each hex digit = 4 binary digits
FF = 1111 1111 = 255

<span class="hl">Bits and Bytes:</span>
1 byte = 8 bits
1 KB = 1,024 bytes ≈ 10³
1 MB = 1,024 KB ≈ 10⁶
1 GB = 1,024 MB ≈ 10⁹

<span class="hl">Overflow:</span>  If a value exceeds what n bits can store, overflow error occurs.
With 8 bits: max unsigned = 255. 256 → wraps to 0.
</div><div class="callout"><div class="callout-lbl">Data Compression</div><ul><li><strong>Lossless:</strong> No data lost. Original can be perfectly reconstructed. Used for text, code, ZIP files. (Run-length encoding: AAABBCC → 3A2B2C).</li><li><strong>Lossy:</strong> Some data permanently removed. Smaller file, cannot recover original. Used for images (JPEG), audio (MP3), video (MP4). Human perception exploited — removes data we don't notice.</li><li><strong>Metadata:</strong> Data about data. A photo's metadata: date, GPS location, camera model, resolution. Metadata can reveal more than the data itself (NSA phone metadata controversy).</li></ul></div><div class="quiz-section">

<h3 class="sub"><span class="sub-icon">💻</span>AP Exam Pseudocode — Official Syntax</h3><div class="code-block"><span class="cmt">// Variables & Assignment</span>
<span class="kw">x</span> ← 5                    <span class="cmt">// Assign value 5 to x</span>
<span class="kw">a</span> ← <span class="kw">b</span>                    <span class="cmt">// Assign value of b to a</span>

<span class="cmt">// Display</span>
<span class="hl">DISPLAY</span>(expression)       <span class="cmt">// Print to screen</span>
<span class="hl">INPUT</span>()                   <span class="cmt">// Read user input</span>

<span class="cmt">// Conditionals</span>
<span class="kw">IF</span> (condition)
{
  &lt;statements&gt;
}
<span class="kw">IF</span> (condition)
{
  &lt;statements&gt;
}
<span class="kw">ELSE</span>
{
  &lt;statements&gt;
}

<span class="cmt">// Loops</span>
<span class="kw">REPEAT</span> n <span class="kw">TIMES</span>
{
  &lt;statements&gt;
}
<span class="kw">REPEAT UNTIL</span> (condition)
{
  &lt;statements&gt;
}

<span class="cmt">// Lists — 1-INDEXED (index starts at 1, NOT 0!)</span>
myList ← [1, 2, 3]
myList[1]              <span class="cmt">// First element = 1 (NOT index 0!)</span>
<span class="hl">LENGTH</span>(myList)         <span class="cmt">// Number of elements</span>
<span class="hl">APPEND</span>(myList, val)    <span class="cmt">// Add val to end</span>
<span class="hl">INSERT</span>(myList, i, val) <span class="cmt">// Insert at index i</span>
<span class="hl">REMOVE</span>(myList, i)      <span class="cmt">// Remove element at index i</span>

<span class="cmt">// Procedures (Functions)</span>
<span class="kw">PROCEDURE</span> myProc(param1, param2)
{
  &lt;statements&gt;
  <span class="kw">RETURN</span> value
}
result ← myProc(arg1, arg2)
</div><div class="warn"><div class="warn-lbl">AP CSP List Trap — 1-Indexed!</div><ul><li>AP Pseudocode lists start at index <strong>1</strong>, NOT 0.</li><li>myList ← [10, 20, 30] → myList[1] = 10, myList[2] = 20, myList[3] = 30.</li><li>This is the #1 pseudocode mistake on the AP exam — especially if you know Python (0-indexed) or Java (0-indexed).</li><li>LENGTH(myList) returns the count of elements. Last element = myList[LENGTH(myList)].</li></ul></div><h3 class="sub"><span class="sub-icon">💻</span>Algorithms — Key Concepts</h3><div class="card-row"><div class="card ac-top"><h4>Linear (Sequential) Search</h4><p>Check each element one by one. Works on ANY list. O(n) worst case. Simple but slow for large lists.</p></div><div class="card ac-top"><h4>Binary Search</h4><p>Only works on SORTED lists. Compare to middle, eliminate half. O(log n). Much faster — but requires sorted data.</p></div><div class="card ac-top"><h4>Undecidable Problems</h4><p>Some problems cannot be solved by any algorithm — halting problem. No matter how much computing power, there is no algorithm that always correctly answers 'will this program halt?'</p></div><div class="card ac-top"><h4>Heuristics</h4><p>Approximate solutions for problems where exact solution is impractical. Not guaranteed optimal, but good enough in reasonable time. Used in: GPS routing, chess AI, spam filters.</p></div></div><div class="quiz-section">

<div class="q-text">1. Why does packet switching make the internet more fault-tolerant than circuit switching?</div>
<ul class="q-choices"><li class="q-choice" data-correct="false" onclick="pick(this,false,'csp_net_q0')">A) Packets are smaller than circuits</li>
<li class="q-choice" data-correct="true" onclick="pick(this,true,'csp_net_q0')">B) Packets can take multiple routes, so if one path fails, data can reroute through alternative paths</li>
<li class="q-choice" data-correct="false" onclick="pick(this,false,'csp_net_q0')">C) Packet switching is faster</li>

<script>function pick(el, isCorrect, qId) {
  const qItem = el.closest('.q-item');
  if (qItem.classList.contains('answered')) return;
  qItem.classList.add('answered');
  el.classList.add(isCorrect ? 'correct' : 'wrong');
  if (!isCorrect) {
    qItem.querySelectorAll('.q-choice[data-correct="true"]').forEach(c => c.classList.add('show-correct'));
  }
  qItem.querySelectorAll('.q-choice').forEach(c => c.classList.add('picked'));
  const explain = document.getElementById('exp-' + qId);
  if (explain) explain.classList.add('show');
  // update score
  const quiz = el.closest('.quiz-section');
  const answered = quiz.querySelectorAll('.q-item.answered').length;
  const correct = quiz.querySelectorAll('.q-choice.correct').length;
  const total = quiz.querySelectorAll('.q-item').length;
  const scoreEl = quiz.querySelector('.quiz-score');
  if (scoreEl) scoreEl.textContent = correct + '/' + answered + ' correct (' + total + ' total)';
}
function resetQuiz(btn) {
  const quiz = btn.closest('.quiz-section');
  quiz.querySelectorAll('.q-item').forEach(q => {
    q.classList.remove('answered');
    q.querySelectorAll('.q-choice').forEach(c => c.classList.remove('correct','wrong','show-correct','picked'));
    q.querySelectorAll('.q-explain').forEach(e => e.classList.remove('show'));
  });
  const scoreEl = quiz.querySelector('.quiz-score');
  if (scoreEl) scoreEl.textContent = '0/0 correct';
}
</script>
