1. What does the following code output?

int[] arr = {5, 3, 8, 1, 9};
int x = 0;
for (int n : arr) { if (n > x) x = n; }
System.out.println(x);
A) 5
B) 9
C) 1
D) 26
B: 9
— This is a max-finding algorithm. x tracks the largest value seen. Starting at 0, it compares each element: 5>0 (x=5), 3>5 false, 8>5 (x=8), 1>8 false, 9>8 (x=9). The for-each loop iterates every element of the array.
2. An ArrayList named 'words' contains ['cat','dog','bird']. After words.remove(1), words contains:
A) ['cat','bird']
B) ['cat','dog']
C) ['dog','bird']
D) ['cat','dog','bird']
A: ['cat','bird']
— remove(int index) removes the element AT that index. Index 1 is 'dog'. After removal: ['cat','bird']. Note: remove(Object o) would remove by value. Since we pass an int literal, it calls the index version. This distinction is a common trap when the list contains Integer objects.

1. What is the value of: "programming".substring(3,7)?
A) "prog"
B) "gram"
C) "gramm"
D) "progr"
B: "gram"
— substring(from, to) returns chars at indices from through to-1 (exclusive end). "programming" indices: p=0,r=1,o=2,g=3,r=4,a=5,m=6,m=7... substring(3,7) gives indices 3,4,5,6 → "gram". The ending index is EXCLUSIVE — common source of off-by-one errors.
2. Which statement correctly creates an ArrayList of Strings?
A) ArrayList list = new ArrayList();
B) ArrayList<String> list = new ArrayList<String>();
C) String[] list = new ArrayList();
D) List list = new String[10];
B is correct.
ArrayLists use generics: ArrayList<Type>. The type parameter tells Java what kind of objects the list holds. Using the raw type (choice A) works but generates a compiler warning and loses type safety. You cannot assign an ArrayList to a String array reference (C) or assign an array to a List without implementation (D).

1. In a properly designed class, instance variables should be declared as:
A) public — so other classes can access them
B) private — to enforce encapsulation, with public getter/setter methods controlling access
C) static — so all instances share them
D) final — so they cannot be changed
B is correct.
Encapsulation (data hiding) is a core OOP principle. Declaring instance variables private prevents direct external access. Getters and setters provide controlled access. This allows the class to validate data (e.g., reject negative balance) and change internal implementation without breaking code that uses the class.

1. An ArrayList contains [10,20,30,40]. After list.add(2, 99), the list contains:
A) [10,20,30,40,99]
B) [10,20,99,30,40]
C) [10,20,99,40]
D) [10,99,20,30,40]
B: [10,20,99,30,40]
— add(int index, E element) INSERTS at the specified index, shifting all existing elements at that index and beyond one position to the right. Index 2 was 30 — now 99 is at index 2, 30 shifts to index 3, 40 shifts to index 4.
2. When iterating an ArrayList and removing elements inside the loop, why is iterating backward preferred?
A) It's faster
B) Going backward ensures removed indices don't affect unprocessed elements (higher indices were already checked)
C) ArrayList doesn't allow forward iteration
D) It prevents a NullPointerException
B is correct.
When you remove at index i while going forward, elements shift left — the element that was at i+1 is now at i and gets skipped on the next iteration. Going backward means you've already processed higher indices, so removals only affect indices you've already examined. Alternatively, decrement i after each remove.

2. Binary search requires that the array be sorted because:
A) It starts from the middle and needs elements in order to decide which half to eliminate
B) It's faster with sorted arrays
C) Java's Arrays.binarySearch requires it
D) It can only search int arrays
A is correct.
Binary search works by comparing the target to the middle element, then eliminating either the left or right half based on whether the target is less or greater. This only works if smaller values are on the left and larger values are on the right — i.e., if the array is sorted. On an unsorted array, binary search gives incorrect results.
