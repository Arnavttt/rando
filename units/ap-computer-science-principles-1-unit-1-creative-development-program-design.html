Creative Development / Program Design — Lesson 5: user interface and accessibility considerations; errors, debugging, and incremental testing

Unit 1 · Lesson 5
Learn on this site

Do not only memorize definitions. Trace execution, predict outputs, explain why the program behaves that way, and then write a smaller version yourself.

user interface and accessibility considerations

user interface and accessibility considerations should be learned by tracing examples and then building your own version. Know the vocabulary, the syntax or pseudocode pattern, what data changes at each step, and the edge cases that could break the program.

Test yourself: teach this idea from memory, give one example, identify one trap, and answer one exam-style question about it.

errors, debugging, and incremental testing

errors, debugging, and incremental testing should be learned by tracing examples and then building your own version. Know the vocabulary, the syntax or pseudocode pattern, what data changes at each step, and the edge cases that could break the program.

Test yourself: teach this idea from memory, give one example, identify one trap, and answer one exam-style question about it.

Objectives
  • Explain and apply user interface and accessibility considerations.
  • Explain and apply errors, debugging, and incremental testing.
How this can be tested
  • trace code or pseudocode
  • identify algorithms and abstractions
  • debug logic or syntax
  • explain data representation or impact
  • write or revise program logic
Practice before moving on
  • Trace a 6–10 line example by hand.
  • Write the smallest possible program that uses the idea.
  • List the edge cases.
  • Explain the algorithm in plain English before coding.
