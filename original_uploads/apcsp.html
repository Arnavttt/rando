💻Binary & Number Systems

Binary (Base-2): Each position: 2⁷ 2⁶ 2⁵ 2⁴ 2³ 2² 2¹ 2⁰ Value: 128 64 32 16 8 4 2 1 Example: 10110101 = 128+32+16+4+1 = 181 Hexadecimal (Base-16): 0-9, A=10, B=11, C=12, D=13, E=14, F=15 Each hex digit = 4 binary digits FF = 1111 1111 = 255 Bits and Bytes: 1 byte = 8 bits 1 KB = 1,024 bytes ≈ 10³ 1 MB = 1,024 KB ≈ 10⁶ 1 GB = 1,024 MB ≈ 10⁹ Overflow: If a value exceeds what n bits can store, overflow error occurs. With 8 bits: max unsigned = 255. 256 → wraps to 0.
Data Compression
  • Lossless: No data lost. Original can be perfectly reconstructed. Used for text, code, ZIP files. (Run-length encoding: AAABBCC → 3A2B2C).
  • Lossy: Some data permanently removed. Smaller file, cannot recover original. Used for images (JPEG), audio (MP3), video (MP4). Human perception exploited — removes data we don't notice.
  • Metadata: Data about data. A photo's metadata: date, GPS location, camera model, resolution. Metadata can reveal more than the data itself (NSA phone metadata controversy).
📝 Unit Quiz 0/0 correct
1. Convert binary 1101 to decimal.
  • A) 11
  • B) 13
  • C) 14
  • D) 15
B: 13. 1101: 1×8 + 1×4 + 0×2 + 1×1 = 8+4+0+1 = 13. Always write out the place values: 8,4,2,1 for a 4-bit number.
2. Which is an example of LOSSY compression?
  • A) ZIP file
  • B) Text file encoding
  • C) MP3 audio compression
  • D) Run-length encoding
C: MP3. MP3 permanently removes audio data that human hearing is less sensitive to (high frequencies, sounds masked by louder sounds). The original audio cannot be recovered. ZIP (A) is lossless — your files are perfectly recovered.

💻AP Exam Pseudocode — Official Syntax

// Variables & Assignment x ← 5 // Assign value 5 to x a ← b // Assign value of b to a // Display DISPLAY(expression) // Print to screen INPUT() // Read user input // Conditionals IF (condition) { <statements> } IF (condition) { <statements> } ELSE { <statements> } // Loops REPEAT n TIMES { <statements> } REPEAT UNTIL (condition) { <statements> } // Lists — 1-INDEXED (index starts at 1, NOT 0!) myList ← [1, 2, 3] myList[1] // First element = 1 (NOT index 0!) LENGTH(myList) // Number of elements APPEND(myList, val) // Add val to end INSERT(myList, i, val) // Insert at index i REMOVE(myList, i) // Remove element at index i // Procedures (Functions) PROCEDURE myProc(param1, param2) { <statements> RETURN value } result ← myProc(arg1, arg2)
AP CSP List Trap — 1-Indexed!
  • AP Pseudocode lists start at index 1, NOT 0.
  • myList ← [10, 20, 30] → myList[1] = 10, myList[2] = 20, myList[3] = 30.
  • This is the #1 pseudocode mistake on the AP exam — especially if you know Python (0-indexed) or Java (0-indexed).
  • LENGTH(myList) returns the count of elements. Last element = myList[LENGTH(myList)].

💻Algorithms — Key Concepts

Linear (Sequential) Search

Check each element one by one. Works on ANY list. O(n) worst case. Simple but slow for large lists.

Binary Search

Only works on SORTED lists. Compare to middle, eliminate half. O(log n). Much faster — but requires sorted data.

Undecidable Problems

Some problems cannot be solved by any algorithm — halting problem. No matter how much computing power, there is no algorithm that always correctly answers 'will this program halt?'

Heuristics

Approximate solutions for problems where exact solution is impractical. Not guaranteed optimal, but good enough in reasonable time. Used in: GPS routing, chess AI, spam filters.

📝 Unit Quiz 0/0 correct
1. In AP Pseudocode, myList ← [5, 10, 15, 20]. What is myList[3]?
  • A) 5
  • B) 10
  • C) 15
  • D) 20
C: 15. AP Pseudocode is 1-indexed. myList[1]=5, myList[2]=10, myList[3]=15, myList[4]=20. This trips up students who know Python (0-indexed). On the exam, always remember: AP Pseudocode starts at 1.
2. Binary search REQUIRES that the data be:
  • A) Stored in a 2D array
  • B) Sorted
  • C) Numerical
  • D) Larger than 100 elements
B: Sorted. Binary search works by comparing the target to the middle element and eliminating half the list. This only works if the data is sorted — otherwise you can't know which half to keep.

💻How the Internet Works

ConceptDefinitionKey Detail
IP AddressUnique identifier for every device on a networkIPv4: 32-bit (4 octets, e.g. 192.168.1.1). IPv6: 128-bit (more addresses)
DNSDomain Name System — translates URLs to IP addresses'Phone book of the internet': google.com → 142.250.80.46
HTTP/HTTPSProtocol for web communicationHTTPS uses TLS/SSL encryption. HTTP is unencrypted.
TCP/IPTransmission Control Protocol / Internet ProtocolTCP ensures reliable, ordered delivery. Packets may take different routes.
UDPUser Datagram ProtocolFaster but unreliable (no delivery guarantee). Used for video streaming, gaming.
Packet SwitchingData split into packets, sent independently, reassembledEach packet may take a different path. More efficient and fault-tolerant than circuit switching.
RedundancyMultiple paths between nodes in a networkIf one path fails, data reroutes. The internet was designed for this resilience.
📝 Unit Quiz 0/0 correct
1. Why does packet switching make the internet more fault-tolerant than circuit switching?
  • A) Packets are smaller than circuits
  • B) Packets can take multiple routes, so if one path fails, data can reroute through alternative paths
  • C) Packet switching is faster
  • D) Packets are encrypted by default
B is correct. In circuit switching (like old telephone calls), one dedicated path is established for the entire communication — if it breaks, the call drops. Packet switching breaks data into independently routed packets, so network failures only affect some packets, which can be rerouted or re-requested.
Create PT
Performance Task — 6-Row Rubric
30% of AP Score

💻Create PT Requirements

What You Submit
  • 1. Program code — your program file(s). Must include at least one procedure you wrote.
  • 2. Video — 1 minute max showing your program running and its input/output.
  • 3. Written responses — 4 prompts (3A–3D) submitted in the AP Digital Portfolio.

💻The 6-Row Rubric — Row by Row

RowPointsWhat to Include
Row 1: Program Purpose & Function1Describe what the program does (purpose), what it accepts as input, and what it produces as output. Reference the VIDEO.
Row 2: Data Abstraction1Show a LIST being used to store/access data. Explain how the list stores multiple values and why a list was necessary vs. individual variables.
Row 3: Managing Complexity1Explain how the list MANAGES COMPLEXITY — how it would be more complex without the list. Don't just say it stores data — explain what would change in the code without it.
Row 4: Procedural Abstraction1Show a PROCEDURE with a parameter that affects functionality. Explain what the procedure does, what the parameter represents, and how it's called.
Row 5: Algorithm Implementation1Show an ALGORITHM inside your procedure that includes: SEQUENCING + SELECTION (if/conditional) + ITERATION (loop). Trace through the algorithm with a specific input.
Row 6: Testing1Describe TWO DIFFERENT test cases with specific inputs, expected outputs, and actual outputs. Tests should check different conditions (e.g., edge case vs. normal case).
Create PT Mistakes That Lose Points
  • Row 2: Showing a list but using it just as a display — the list must actually store values that the program uses for functionality.
  • Row 3: 'The list stores all the values' — not enough. Explain what CODE would look like without the list (parallel variables) and why that increases complexity.
  • Row 4: Calling a built-in function (DISPLAY, APPEND) doesn't count. Must be a STUDENT-CREATED procedure with a PARAMETER.
  • Row 5: The algorithm must have all three: sequence, selection, AND iteration. One missing = 0.
  • Row 6: Two test cases with the SAME condition = 0. Tests must cover DIFFERENT scenarios.
📝 Unit Quiz 0/0 correct
1. For Row 3 (Managing Complexity), the strongest response explains that the list:
  • A) Makes the code look cleaner
  • B) Stores multiple values
  • C) Reduces code complexity — without it, the program would require multiple separate variables or conditional chains, making the code harder to maintain and expand
  • D) Is required by the assignment
C is correct. Row 3 is specifically about MANAGING COMPLEXITY — not just using a list. The rubric requires you to explain how the code would be MORE COMPLEX without the list. 'I would need 20 separate variables instead of 1 list, and my loop would need to be rewritten for each one' is the kind of explanation that earns the point.
Perfect Score Checklist
  • AP Pseudocode lists are 1-INDEXED — myList[1] is the first element
  • Binary = base 2. Hex = base 16 (A=10 through F=15). Each hex digit = 4 bits.
  • Lossless = perfect recovery (ZIP, PNG). Lossy = data permanently removed (MP3, JPEG).
  • Metadata = data about data — can reveal location, time, identity
  • Binary search only works on SORTED data
  • Undecidable problems cannot be solved by any algorithm (halting problem)
  • Packet switching: data split into independently routed packets → fault tolerant
  • DNS translates domain names to IP addresses
  • HTTPS = encrypted (TLS). HTTP = unencrypted.
  • Create PT Row 5 requires ALL THREE: sequence + selection + iteration
  • Create PT Row 6 requires TWO tests with DIFFERENT conditions
