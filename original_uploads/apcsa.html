Common FRQ Traps
  • ArrayList is 0-indexed. Valid indices: 0 to list.size()-1
  • When removing elements from an ArrayList while iterating FORWARD, indices shift — iterate backward or decrement counter after remove
  • set(i,x) replaces at index i. add(i,x) inserts at index i (shifts elements right). Know which you need.
  • 2D array: arr[row][col]. arr.length = number of rows. arr[0].length = number of columns.
  • String.equals() not ==. Integer wrapper objects use .equals() too.
  • Math.random() returns [0.0, 1.0) — to get int 0-9: (int)(Math.random() * 10)
📝 Unit Quiz 0/0 correct
1. What does the following code output? int[] arr = {5, 3, 8, 1, 9}; int x = 0; for (int n : arr) { if (n > x) x = n; } System.out.println(x);
  • A) 5
  • B) 9
  • C) 1
  • D) 26
B: 9 — This is a max-finding algorithm. x tracks the largest value seen. Starting at 0, it compares each element: 5>0 (x=5), 3>5 false, 8>5 (x=8), 1>8 false, 9>8 (x=9). The for-each loop iterates every element of the array.
2. An ArrayList named 'words' contains ['cat','dog','bird']. After words.remove(1), words contains:
  • A) ['cat','bird']
  • B) ['cat','dog']
  • C) ['dog','bird']
  • D) ['cat','dog','bird']
A: ['cat','bird'] — remove(int index) removes the element AT that index. Index 1 is 'dog'. After removal: ['cat','bird']. Note: remove(Object o) would remove by value. Since we pass an int literal, it calls the index version. This distinction is a common trap when the list contains Integer objects.

☕Primitives vs. Objects

Primitive Types (3 tested)

int — whole numbers. double — decimals. boolean — true/false. Stored by VALUE. Default: int=0, double=0.0, boolean=false.

Reference Types (Objects)

String, Integer, ArrayList, custom classes. Variables store a REFERENCE (address) to the object in memory. Default: null.

Type Casting

(int)(3.7) → 3 (truncates). (double)5/2 → 2.5. Integer/int division truncates: 5/2 → 2. To force double: cast ONE operand.

Wrapper Classes

Integer and Double wrap primitives for use in ArrayLists. Autoboxing: int→Integer automatically. Unboxing: Integer→int automatically.

Step-by-Step: Reverse a String
1
Create an empty String result = "";
2
Loop i from s.length()-1 DOWN to 0 (inclusive).
3
Each iteration: result += s.substring(i, i+1); (or result += s.charAt(i);)
4
Return result.
📝 Unit Quiz 0/0 correct
1. What is the value of: "programming".substring(3,7)?
  • A) "prog"
  • B) "gram"
  • C) "gramm"
  • D) "progr"
B: "gram" — substring(from, to) returns chars at indices from through to-1 (exclusive end). "programming" indices: p=0,r=1,o=2,g=3,r=4,a=5,m=6,m=7... substring(3,7) gives indices 3,4,5,6 → "gram". The ending index is EXCLUSIVE — common source of off-by-one errors.
2. Which statement correctly creates an ArrayList of Strings?
  • A) ArrayList list = new ArrayList();
  • B) ArrayList<String> list = new ArrayList<String>();
  • C) String[] list = new ArrayList();
  • D) List list = new String[10];
B is correct. ArrayLists use generics: ArrayList<Type>. The type parameter tells Java what kind of objects the list holds. Using the raw type (choice A) works but generates a compiler warning and loses type safety. You cannot assign an ArrayList to a String array reference (C) or assign an array to a List without implementation (D).

☕Writing a Complete Class — FRQ 2 Pattern

How to Write a Class from Scratch
1
Declare instance variables (private). Match the types to what the constructor receives.
2
Write the constructor: public ClassName(params) { this.var = param; ... }
3
Write accessor methods (getters): public Type getVar() { return var; }
4
Write mutator methods (setters): public void setVar(Type val) { this.var = val; }
5
Write behavior methods: use instance variables and parameters to compute results.
6
toString() method (if asked): public String toString() { return 'ClassName[var=' + var + ']'; }
// FRQ 2 Template — complete class public class BankAccount { // Step 1: private instance variables private String owner; private double balance; // Step 2: constructor public BankAccount(String owner, double initialBalance) { this.owner = owner; this.balance = initialBalance; } // Step 3: accessor public double getBalance() { return balance; } // Step 4: mutator public void deposit(double amount) { if (amount > 0) balance += amount; } // Step 5: behavior method public boolean withdraw(double amount) { if (amount > 0 && amount <= balance) { balance -= amount; return true; } return false; } }
📝 Unit Quiz 0/0 correct
1. In a properly designed class, instance variables should be declared as:
  • A) public — so other classes can access them
  • B) private — to enforce encapsulation, with public getter/setter methods controlling access
  • C) static — so all instances share them
  • D) final — so they cannot be changed
B is correct. Encapsulation (data hiding) is a core OOP principle. Declaring instance variables private prevents direct external access. Getters and setters provide controlled access. This allows the class to validate data (e.g., reject negative balance) and change internal implementation without breaking code that uses the class.

☕ArrayList — Key Operations

import java.util.ArrayList; ArrayList<String> list = new ArrayList<String>(); list.add("apple"); // add to end list.add(1, "banana"); // INSERT at index 1 — shifts elements right list.get(0); // → "apple" — does NOT remove list.set(0, "pear"); // REPLACE index 0 with "pear" list.remove(0); // remove at index 0 — shifts left list.size(); // number of elements // SAFE removal while iterating — go BACKWARD for (int i = list.size()-1; i >= 0; i--) { if (list.get(i).equals("remove me")) list.remove(i); // safe — only affects indices we've already passed }

2

3

4
Check condition: if(item satisfies condition)
5
Add to result: result.add(item);

Return result after loop ends.
📝 Unit Quiz 0/0 correct
1. An ArrayList contains [10,20,30,40]. After list.add(2, 99), the list contains:
  • A) [10,20,30,40,99]
  • B) [10,20,99,30,40]
  • C) [10,20,99,40]
  • D) [10,99,20,30,40]
B: [10,20,99,30,40] — add(int index, E element) INSERTS at the specified index, shifting all existing elements at that index and beyond one position to the right. Index 2 was 30 — now 99 is at index 2, 30 shifts to index 3, 40 shifts to index 4.
2. When iterating an ArrayList and removing elements inside the loop, why is iterating backward preferred?
  • A) It's faster
  • B) Going backward ensures removed indices don't affect unprocessed elements (higher indices were already checked)
  • C) ArrayList doesn't allow forward iteration
  • D) It prevents a NullPointerException
B is correct. When you remove at index i while going forward, elements shift left — the element that was at i+1 is now at i and gets skipped on the next iteration. Going backward means you've already processed higher indices, so removals only affect indices you've already examined. Alternatively, decrement i after each remove.

☕2D Arrays — FRQ 4 Pattern

// Declare and create a 2D array int[][] grid = new int[3][4]; // 3 rows, 4 cols int[][] grid2 = {{1,2},{3,4},{5,6}}; // initializer grid.length // → 3 (number of ROWS) grid[0].length // → 4 (number of COLS) grid[r][c] // element at row r, col c // Standard row-major traversal (row by row) for (int r = 0; r < grid.length; r++) { for (int c = 0; c < grid[r].length; c++) { System.out.print(grid[r][c] + " "); } } // Column-major traversal (col by col) for (int c = 0; c < grid[0].length; c++) { for (int r = 0; r < grid.length; r++) { System.out.print(grid[r][c] + " "); } }
Step-by-Step: Find Max in 2D Array
1
Initialize max = grid[0][0] (first element as initial comparison)
2
Nested loops: outer over rows (r from 0 to grid.length-1), inner over cols (c from 0 to grid[r].length-1)
3
Each iteration: if(grid[r][c] > max) { max = grid[r][c]; }
4
After loops: return max

📝 Unit Quiz 0/0 correct
1. A 2D array grid has 4 rows and 6 columns. What is the value of grid[0].length?
  • A) 4
  • B) 6
  • C) 24
  • D) 0
B: 6 — grid.length is the number of rows (4). grid[0].length is the number of columns in row 0 (6). This is the most important 2D array fact: length gives rows, [0].length gives columns. Always use grid[r].length in the inner loop condition to handle jagged arrays safely.
2. Binary search requires that the array be sorted because:
  • A) It starts from the middle and needs elements in order to decide which half to eliminate
  • B) It's faster with sorted arrays
  • C) Java's Arrays.binarySearch requires it
  • D) It can only search int arrays
A is correct. Binary search works by comparing the target to the middle element, then eliminating either the left or right half based on whether the target is less or greater. This only works if smaller values are on the left and larger values are on the right — i.e., if the array is sorted. On an unsorted array, binary search gives incorrect results.

  • 2D array: grid.length = rows, grid[0].length = columns

  • Remove from ArrayList while looping: go backward or decrement counter after remove()

  • Math.random() returns [0.0, 1.0). For int 0 to n-1: (int)(Math.random()*n)
